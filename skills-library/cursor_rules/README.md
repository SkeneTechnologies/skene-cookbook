# Tool-Specific Coding Guidelines

This directory contains coding guidelines and best practices for specific tools, frameworks, and platforms. These are curated from various sources including awesome-cursor-rules and other community collections.

## Organization

Skills are organized by tool/framework name:
- **Cloud platforms**: AWS ECS, and more
- **Databases**: PostgreSQL, MongoDB, etc.
- **Frameworks**: React, Django, FastAPI, TensorFlow, etc.
- **AI/ML Tools**: AutoGen, CrewAI, etc.
- **DevOps tools**: Docker, Kubernetes, Terraform, etc.
- **Testing tools**: Pytest, Jest, Playwright, etc.
- **Communication**: Microsoft Teams, Slack, etc.
- **Python Libraries**: Click, Typer, Stripe, etc.

## Purpose

These skills provide context-specific coding conventions, best practices, and common patterns for working with specific tools and frameworks. They are designed to help maintain consistency and quality when working with these technologies.

## Usage

These skills can be manually invoked in Claude Code when working with a specific tool, or you can reference them when you need tool-specific guidance.

## Sources

- awesome-cursor-rules community collection
- Official framework documentation
- Community best practices
- Expert practitioner contributions

## Note

These are tool-specific guidelines rather than general-purpose skills. They complement the broader skills library by providing detailed, focused guidance for specific technologies.
